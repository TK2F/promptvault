# Contributing to PromptVault

Thank you for your interest in contributing to PromptVault!

## How to Contribute

### Reporting Issues

- Use the GitHub Issues page
- Describe the problem clearly
- Include browser version and OS
- Provide steps to reproduce

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

### Code Style

- Follow existing code patterns
- Use TypeScript
- Run `npm run lint` before committing
- Use meaningful commit messages

### Development Setup

```bash
npm install
npm run dev
```

### Building

```bash
npm run build
```

# コントリビューションガイド

PromptVault へのご貢献に感謝します！

## 貢献方法

### 問題の報告

- GitHub Issues を使用してください
- 問題を明確に記述してください
- ブラウザバージョンとOSを含めてください
- 再現手順を提供してください

### プルリクエスト

1. リポジトリをフォーク
2. フィーチャーブランチを作成 (`git checkout -b feature/amazing-feature`)
3. 変更をコミット (`git commit -m 'Add amazing feature'`)
4. ブランチにプッシュ (`git push origin feature/amazing-feature`)
5. プルリクエストを開く

### コードスタイル

- 既存のコードパターンに従う
- TypeScript を使用
- コミット前に `npm run lint` を実行
- 意味のあるコミットメッセージを使用

### 開発セットアップ

```bash
npm install
npm run dev
```

### ビルド

```bash
npm run build
```

## ライセンス

貢献により、あなたの貢献物が MIT ライセンスの下でライセンスされることに同意したものとみなされます。
