---

## 🚀 1. 新規プロジェクトのGitHubアップロード

新しいプロジェクトをGitHubに作りたい時にそのままコピーして使えます。

**依頼テンプレート:**
> このプロジェクトを、GitHub CLIを使って `[リポジトリ名]` という名前のPublicリポジトリとして作成し、最初のコミットとプッシュまで一気にお願いします。

**Antigravityが実行する内容:**
- `git init`, `git add .`, `git commit`
- `gh repo create` (リポジトリ作成)
- `git remote add`, `git push`

## 🆙 2. 修正内容の反映（バージョンアップ）

コードを書き換えた後に、変更をGitHubに反映させたい時のテンプレートです。

**依頼テンプレート:**
> 今の修正をバージョン `[v1.x.x]` としてGitHubに反映してください。CHANGELOGとmanifest.jsonのバージョン更新、Gitコミット、プッシュ、およびリリースタグの作成まで含めてお願いします。

**Antigravityが実行する内容:**
- 関連ファイルのバージョン番号書き換え
- `git commit` & `git push`
- `git tag` の作成とプッシュ

## 📦 3. リリース用ZIPファイルの作成

**依頼テンプレート:**

**Antigravityが実行する内容:**
- `npm run build`
- PowerShellコマンドによるZIP圧縮

## 💡 効率的に依頼するためのヒント

1. **「GitHub CLI (gh)」を推奨する**:
   Antigravityはブラウザ操作ではなく「コマンド（ghコマンド）」を通じてGitHubと対話するのが最も得意です。
2. **「確認なしで進めて」と伝える**:
   「安全だと判断できるコマンドは、私の承認を待たずに提案・実行してください」と伝えると、よりスピーディーに作業が進みます。
3. **具体的なプロジェクト名・リポジトリ名を指定する**:
   「いつもの通り」ではなく、明示的に名前を出すことでミスを防げます。
4. **成果物の確認を求める**:
   「作業が終わったら、GitHubのURLを教えてください」と伝えると、最終的なリンクまで提供します。
